# Post Style Guide

이 문서는 기존 포스트를 읽기 쉽게 재구성할 때 사용하는 기준이다.

## 공통 원칙

1. 본문에 `#` 제목을 다시 쓰지 않는다.
2. 글 시작 3~5줄 안에 `핵심 요약`을 둔다.
3. 본문 큰 구간은 `##`, 하위 설명은 `###`까지만 사용한다.
4. `------` 같은 구분선은 꼭 필요한 경우가 아니면 쓰지 않는다.
5. 코드 블록 뒤에는 반드시 `무엇을 보여 주는 코드인지`를 1~3줄로 설명한다.
6. 글 마지막은 `정리`와 `참고`로 닫는다.
7. 표는 비교가 필요한 경우에만 쓰고, 나열형 정보는 리스트로 푼다.
8. 비교표는 빠르게 스캔할 수 있도록 `✅`, `⚠️`, `❌`, `🚀` 같은 이모지를 제한적으로 사용할 수 있다.

## 권장 구조

### 개념 정리형

1. 핵심 요약
2. 왜 중요한가
3. 핵심 개념 설명
4. 문제가 생기는 패턴 또는 예시
5. 실무에서 줄이는 방법
6. 정리
7. 참고

### 비교형

1. 핵심 요약
2. 한눈에 비교
3. 항목별 설명
4. 언제 무엇을 쓸지
5. 정리
6. 참고

### 절차형

1. 핵심 요약
2. 준비물 또는 전제 조건
3. 단계별 진행
4. 확인 방법
5. 자주 생기는 예외
6. 정리

### 프로젝트 로그형

1. 이번 작업 목표
2. 작업 전 상태
3. 진행한 일
4. 막혔던 점
5. 결과
6. 다음 작업

## 편집 체크리스트

- 제목과 첫 문단만 읽어도 글의 결론이 보이는가
- 섹션 제목만 훑어도 흐름이 이해되는가
- 코드 예시가 설명보다 먼저 길게 나오지 않는가
- 한 문단이 너무 길면 2~3문단으로 나눌 수 있는가
- 같은 의미의 표현이 반복되지 않는가
- 이미지가 있다면 이미지 앞뒤에 맥락 설명이 있는가

## 우선 정리 대상

1. 최근 기술 글
2. 조회 가능성이 높은 비교/최적화 글
3. 검색 유입용 Git / C# / Unity 지식 글
4. 프로젝트 로그
